# AGENTS

# Product Identity

Canopy is a local-first personal intelligence and contextual memory system.

The product is NOT:
- a therapy app,
- a dating CRM,
- a productivity-gamification platform,
- or a personality scoring engine.

The product IS:
- a cognitive offloading system,
- contextual memory infrastructure,
- and lightweight decision support platform.

---

# Architectural Priorities

Priority order:
1. Low-friction capture
2. Privacy
3. Retrieval quality
4. Sparse high-signal data
5. Human interpretability
6. Local-first reliability

---

# Development Principles

Prefer:
- simplicity
- modularity
- progressive enhancement
- local processing
- explainable behavior

Avoid:
- unnecessary microservices
- premature optimization
- opaque scoring
- notification overload
- surveillance-like UX

---

# AI Usage

AI should:
- summarize
- assist retrieval
- identify recurring themes
- reduce cognitive effort

AI should NOT:
- diagnose people
- generate deterministic conclusions
- manipulate user decisions

---

# UX Philosophy

The application should feel:
- calm
- sparse
- reflective
- trustworthy
- minimally intrusive

Avoid:
- dopamine loops
- social media dynamics
- excessive dashboards
- productivity anxiety
