# Ontology

# Core Entity Types

## Person

Represents:
- colleagues
- friends
- collaborators
- recurring contacts

Store:
- interactions
- recurring observations
- context-linked patterns

Avoid fixed identity judgments.

---

## Interaction

Represents:
- conversations
- meetings
- commitments
- operational exchanges
- emotionally significant events

Fields:
- timestamp
- participants
- context
- observation
- outcome
- confidence

---

## Observation

Observations should remain:
- contextual,
- behavioral,
- and revisable.

Preferred:
- "frequently delays commitments under pressure"

Avoid:
- "unreliable person"

---

## Task / Responsibility

Represents:
- recurring work
- invisible ownership
- maintenance burden
- delegation risk
- strategic value

Potential dimensions:
- urgency
- reversibility
- visibility
- effort
- growth value
- operational cost

---

# Pattern Types

Examples:
- initiative imbalance
- recurring ambiguity
- invisible labor accumulation
- repeated cancellation loops
- operational dependency
- energy drain patterns

---

# Confidence and Decay

All interpretations should support:
- uncertainty
- confidence scoring
- historical revision
- contextual expiration
